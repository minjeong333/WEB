

동적페이지 만들기 (프로그램 - 자바 - 상속받아서 만드는 자바 프로그램 - HttpServlet)
1. HttpServlet상속받은 클래스 작성하기
2. 약속된 매서드(service) 즉, 서비스할 내용을 오버라이드 하기
3. 서비스할 비지니스 로직은 별도의 자바 파일로 분리하기 
4. 서블릿 등록하기, url-pattern 지정하기 - 중복되지 않기 !! ∵ 패턴하나가 하나의 서비스를 의미




웹애플리케이션

웹서버 (응답하는 쪽) - 클라이언트 (요청하는 쪽)

HttpServlet
HttpServletRequest (요청에 관련된 정보)
HttpServletResponse (응답에 관련된 정보)


     요청                               응답      
	-Get 							  -Get (서버가 가지고 있는 정보를 단지 제공해주는 서비스)
	-Post							  -Post
	
	
요청을 구분한 예시
get  : a태그, url직접요청, form태그 method="get"   => 서버는 get 요청을 인식

post : form 태그 method="post"				  => 서버는 post요청으로 인식

get요청 : 사용자가 보내는 데이터를 헤더에 담는다. url노출, 데이터의 형식, 크기가 정해져있다
post요청 : 사용자가 보내는 데이터를 요청 바디에 담는다. 노출되지 않음, 데이터형식, 크기가 제한이 없다
 

요청을 구분하는 이유
효율적인 서버의 응대
get  : 가벼운 요청
post : 무거운 요청


service   ->    request요청 정보를 확인    ->  get -> doGet,   post => doPost 요청해줌

요청을 구분해서 요청을 처리하는 게 일반적임!!
